# 사용자 시나리오 및 테스트 케이스 (User Scenarios & Test Cases)

**작성자**: Business Analyst
**문서 버전**: 1.0
**참고 문서**: `2-prd.md`

이 문서는 '함월' 시스템의 주요 기능을 사용자 관점에서 검증하기 위한 시나리오와 테스트 케이스를 정의합니다.

---

## 1. 회원 가입 및 로그인 (Authentication)

### 시나리오 1: 신규 회원 가입
*   **Actor**: 비회원 (잠재적 회원)
*   **Pre-condition**: 로그인되지 않은 상태
*   **Steps**:
    1.  메인 페이지 접속 후 '회원가입' 버튼 클릭.
    2.  이메일, 비밀번호, 이름, 전화번호 입력.
    3.  '가입하기' 버튼 클릭.
*   **Expected Result**:
    *   별도의 승인 대기 없이 즉시 가입 완료 처리.
    *   자동으로 로그인되며 메인 대시보드로 이동.
    *   기본 역할은 'MEMBER(일반 회원)'로 설정됨.

### 시나리오 2: 로그인 및 로그아웃
*   **Actor**: 회원
*   **Steps**:
    1.  로그인 페이지에서 이메일/비밀번호 입력 후 로그인 시도.
    2.  로그인 성공 시 대시보드 진입.
    3.  우측 상단 프로필 메뉴에서 '로그아웃' 클릭.
*   **Expected Result**:
    *   로그인 성공 시 사용자 이름이 표시됨.
    *   로그아웃 시 로그인 페이지로 리다이렉트됨.

---

## 2. 역할 관리 (Role Management)

### 시나리오 3: 관리자 권한 부여 (Admin Only)
*   **Actor**: 회장 (Super Admin)
*   **Pre-condition**: '회장' 계정으로 로그인 상태
*   **Steps**:
    1.  '회원 명부' 메뉴 이동.
    2.  특정 회원(예: 홍길동)을 선택하여 상세 정보 조회.
    3.  역할 변경 드롭다운에서 '총무(ADMIN)' 선택 후 저장.
*   **Expected Result**:
    *   홍길동의 역할이 'ADMIN'으로 변경됨.
    *   홍길동이 로그인하면 '회칙 관리', '장부 관리'의 쓰기 권한이 활성화됨.

### 시나리오 4: 일반 회원의 권한 접근 제한
*   **Actor**: 일반 회원
*   **Steps**:
    1.  '회원 명부' 메뉴 이동.
    2.  다른 회원의 정보 클릭.
*   **Expected Result**:
    *   회원 정보 조회는 가능하나, '역할 변경' 버튼이나 기능이 보이지 않아야 함.

---

## 3. 회칙 관리 (Bylaws)

### 시나리오 5: 회칙 개정 (Admin Only)
*   **Actor**: 관리자 (회장/총무)
*   **Steps**:
    1.  '회칙 관리' 메뉴 이동.
    2.  '새 회칙 등록' 버튼 클릭.
    3.  버전명(예: 2024-2차), 시행일, 변경 사유, 본문 내용 입력.
    4.  '등록' 버튼 클릭.
*   **Expected Result**:
    *   새로운 회칙이 '현재 유효한 회칙'으로 등록됨.
    *   기존 회칙은 '과거 이력'으로 이동됨.

### 시나리오 6: 회칙 이력 조회
*   **Actor**: 일반 회원
*   **Steps**:
    1.  '회칙' 메뉴 이동.
    2.  기본적으로 현재 회칙이 보임.
    3.  '개정 이력' 탭 클릭.
    4.  과거 버전 리스트 중 하나 클릭.
*   **Expected Result**:
    *   과거 회칙의 내용과 당시 변경 사유가 조회됨.

---

## 4. 장부 관리 (Ledger)

### 시나리오 7: 입출금 내역 등록 (Admin Only)
*   **Actor**: 관리자
*   **Steps**:
    1.  '재정 관리' 메뉴 이동.
    2.  '거래 등록' 버튼 클릭.
    3.  날짜, 유형(지출), 금액(50,000원), 카테고리(식대), 비고(회식비) 입력.
    4.  '저장' 버튼 클릭.
*   **Expected Result**:
    *   거래 내역 리스트 최상단에 추가됨.
    *   전체 잔액이 50,000원 차감되어 갱신됨.

### 시나리오 8: 마이너스 잔액 테스트
*   **Actor**: 관리자
*   **Pre-condition**: 현재 잔액 10,000원
*   **Steps**:
    1.  30,000원 지출 내역 등록.
*   **Expected Result**:
    *   에러 없이 등록 성공.
    *   잔액이 -20,000원으로 표시됨 (빨간색 등으로 강조 가능).

### 시나리오 9: 장부 조회
*   **Actor**: 일반 회원
*   **Steps**:
    1.  '재정 관리' 메뉴 이동.
*   **Expected Result**:
    *   전체 거래 내역 리스트가 최신순으로 보임.
    *   '등록/수정' 버튼은 보이지 않음 (읽기 전용).
    *   현재 총 잔액이 정확히 표시됨.

---

## 5. 비기능 테스트 (Non-functional)

### 시나리오 10: 모바일 반응형 확인
*   **Actor**: 모든 사용자
*   **Environment**: 모바일 브라우저 (또는 개발자 도구 모바일 뷰)
*   **Steps**:
    1.  모바일 해상도(예: iPhone SE, Galaxy S 등)에서 메인, 장부, 회칙 페이지 접속.
*   **Expected Result**:
    *   좌우 스크롤 없이 컨텐츠가 화면 폭에 맞춰 잘 보임.
    *   버튼 크기가 터치하기 적절함.
    *   메뉴 네비게이션(햄버거 메뉴 등)이 정상 작동함.
